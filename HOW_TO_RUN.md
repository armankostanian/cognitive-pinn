# Как запустить

Короткое руководство по воспроизведению результатов. Все синтетические проверки
запускаются на CPU за минуты; эксперименты на реальных данных требуют GPU.

## 0. Установка

```bash
git clone https://github.com/armankostanian/cognitive-pinn.git
cd cognitive-pinn
pip install -r requirements.txt
```

Проверка CUDA (для реальных данных):

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
```

## 1. Синтетические проверки (CPU, без GPU)

Запускать из каталога `src/`, чтобы корректно резолвились локальные импорты:

```bash
cd src/

# Замкнутая форма восстановления параметров (теорема об идентифицируемости)
python cpinn_identifiability_K2.py
# Ожидание: относительная ошибка восстановления <= 1e-15 при нулевом шуме.

# Синтетический K=5, сквозной пайплайн
python cpinn_synthetic_pipeline.py
# Ожидание: F1 графа пререквизитов = 1.0; AUC ~ oracle.

# Санити-чек с oracle-параметрами
python cpinn_oracle_check.py
```

## 2. Батчевый решатель и проверка эквивалентности

Медленный путь обучает по одному студенту за раз (узкое место на больших данных).
Батчевый решатель интегрирует весь батч одним вызовом ODE на общей целочисленной
сетке времени. Он численно эквивалентен медленному пути (предсказания бит-в-бит,
градиенты ~1e-10) и на больших данных быстрее в ~245 раз.

```bash
cd src/
python cpinn_batched_test.py
# Ожидание: max|dp| = 0.0; max|d grad| ~ 1e-10; ускорение (на GPU больше, чем на CPU).
```

## 3. Эксперименты на реальных данных (GPU)

Пайплайн подготовки данных (очистка, фолды, baselines DKT/BKT, препроцессинг Junyi,
построение экспертного графа, сверка структуры) ведётся в отдельном пакете и здесь не
дублируется, чтобы не смешивать модель и экспериментальную обвязку.

Ключевые настройки для воспроизводимости (см. также `RESULTS.md`):

- `lambda_ode = 0` — отключает ODE-residual, чтобы objective батчевого и медленного путей
  совпадали; именно с этим значением получены все чистые результаты.
- Для эксперимента по восстановлению структуры дополнительно `lambda_l1 = 0`, `lambda_l2 = 0`
  (абляция): L1-штраф зануляет внедиагональ матрицы A и подавляет искомую структуру.
- `integrator_step = 0.25`.
- Единые student-level фолды для всех моделей; ключи (student, step) согласованы;
  значимость — тест DeLong + кластерный бутстрэп.

Важно про данные: в ASSISTments-2009 есть дублирующиеся строки (Xiong et al., 2016),
завышающие AUC всех моделей. Все прогоны выполняются на очищенных (дедуплицированных)
данных. Сырые логи студентов не распространяются — берутся из стандартных публичных релизов.

## 4. Что считать корректным результатом

- Синтетика: восстановление параметров ~1e-15, F1 графа = 1.0. Если хуже — проблема в среде.
- Реальные данные: CPINN по AUC **ниже** DKT/BKT — это ожидаемо и является документированным
  компромиссом, а не ошибкой. Главный результат — не точность, а внешняя валидация параметров
  (Lambda <-> сложность тем, rho = 0.92). Подробности — в `RESULTS.md`.
